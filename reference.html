Phasing Quick Reference

Day 1 Ready

Work immediately with basic catalog:

  • Auto Complete
  • Part Number Normalization
  • Basic Facets
  • Recent Search
  • Curations
  • Search Sandbox
  • Configuration Management
  • Search Dashboard
  • Zero Results Recovery
  • Analytics (grows over time)

Day 30-60

After behavioral data accumulates:

  • Trending Search
  • Related Searches
  • Query Behaviour Labeller
  • Intelligent Synonyms
  • A/B Testing

Day 90+ / When Data Ready

Require specific integrations:

  • ML Ranking (90+ days behavioral)
  • Recommendations (orders + users)
  • Intelligent Auto Complete
  • Cross-Brand Equivalents (relationships)
  • Recently Ordered (orders)
  • Multi-Layered Relevance

Key Principles

Data is the Ceiling

Don't over-invest when data quality caps value. Fix data first, or accept lower investment level.

Capacity Determines Approach

Low capacity = more self-service investment needed. High capacity = they can fill gaps.

Right-Size to Conditions

Match investment to what's achievable. Premium investment with poor data = wasted effort.

Document Assumptions

Put data quality and capacity assumptions in SOW. Be explicit about what's expected.

Plan for Iteration

MVP now, upgrade later is a valid strategy. Launch, learn, improve.

B2B ≠ B2C

80% of orders are repeats. Optimize for reordering, not just discovery.

Glossary

ELSER
Elastic Learned Sparse Encoder — semantic search model
Facets
Filter options based on product attributes (brand, size, etc.)
Curations
Manual merchandising rules — pin, boost, bury, redirect
Zero Results
Search returning no products — critical to recover from
Behavioral Data
Search logs, clicks, conversions — accumulated over time
Cross-Reference
Mapping between equivalent products across brands
Kitting
Bundling products together as custom assemblies
ML Ranking
Machine learning to optimize result ordering
AI Enrichment
Using AI to generate missing product content
Governance
Ongoing attention needed to maintain feature quality
